​AI Engineer — Ислам
​Роль AI
ИИ-инженер, ответственный за интеллектуальный слой приложения, семантическое обогащение данных и взаимодействие с внешними ИИ-агентами.
​Зоны ответственности:
​Проектирование и реализация AI-слоя: внедрение векторного поиска (pgvector) в БД Postgres.
​Семантический поиск: реализация поиска по турам на основе смысла запроса (Natural Language Search).
​Разработка MCP-сервера: создание tours-mcp для предоставления инструментов (tools) внешним ИИ-агентам.
​Интеграция LLM: пайплайны для суммаризации туров и генерации рекомендаций.
​Обеспечение качества: валидация ответов LLM через Pydantic и проверка точности поиска (Evaluation).
​Синхронизация с бэкендом: тесное взаимодействие с Нурсултаном (Backend) для соблюдения контракта OpenAPI.
​Правила работы:
​Сначала контракт: любые AI-эндпоинты (например, /api/tours/search) должны быть описаны в contracts/openapi.yaml.
​Векторная миграция: добавление полей для эмбеддингов — только через миграции Alembic.
​Безопасность: защита от Prompt Injection и фильтрация чувствительных данных.
​Производительность: генерация тяжелых эмбеддингов не должна блокировать основной поток API.
​Локальный запуск: весь AI-функционал должен подниматься через docker compose up.
​MCP / Инструменты:
​Tours MCP Server (Core Task):
​search_tours — поиск с учетом фильтров и семантики.
​get_tour_details — получение полной информации о туре для контекста LLM.
​create_booking — инструмент для оформления бронирования голосом/текстом.
​Vector Store: использование Postgres + pgvector как основного хранилища знаний.
​Основные задачи AI Engineer:
​Инфраструктура: настройка подключения к провайдерам LLM (OpenAI/Anthropic) или локальным моделям.
​Векторизация: создание сервиса app/services/vector_service.py для обработки описаний туров.
​Сидирование: доработка сидера для автоматической генерации эмбеддингов при запуске.
​Эндпоинты:
​POST /api/tours/semantic-search — поиск по смыслу.
​GET /api/tours/{id}/recommendations — похожие туры.
​Тестирование: pytest для проверки работы векторного поиска и корректности работы MCP-инструментов.
​Архитектура AI-модуля:backend/
  app/
    ai/
      vector_store.py    # Работа с pgvector
      prompts.py         # Системные промпты и шаблоны
      schemas.py         # Pydantic модели для AI ответов
    mcp/
      server.py          # Точка входа MCP сервера
      handlers.py        # Логика инструментов (tools)
Команды Ислама:
​make ai-ingest — векторизовать все туры в базе данных.
​make mcp-dev — запуск MCP-сервера в режиме отладки.
​make test-ai — запуск тестов на качество поиска и ответов.
